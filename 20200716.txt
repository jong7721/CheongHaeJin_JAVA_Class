자바와 다른언어의 차이점
ㄴ자바 - 객체지향 object
ㄴC언어 - 포인터라는 개념 데이터를 만들어서 연산을 처리하는것


메모리 -데이터의 저장공간 관리는 어떻게?
메모리의 주소 번지(어드레스)에 저장 - 포인터

포인터의 주소값을 잘못쓰면 엉뚱한 값을 나타낸다

C언어에서는 포인터 직접적으로 쓰는 방법은 효과적이나 치명적일 수 있다.

반면 오브젝트는 참조형 데이터를 관리한다. 메모리에 생성된 데이터를 접근하는 방법!

해당 객체 오브젝트의 이름을 통해서 간접적으로 접근!

C언어에 비해 java는 위험성을 줄여서 메모리를 관리한다.


클래스의 개념
ㄴ어떤 객체를 정의하는것
ㄴ이것을 프로그램상에 특정한 형태인 type로 옮긴 것이 클래스(class)
ㄴ데이터의 기본 자료형만으로는 현실 세계를 모델링하기에 부족
ㄴ개발자가 원하는 모든 객체의 타이블 정의해서 사용할 수 있게 한다.
ㄴ클래스는 현실 세계의 객체를 데이터를 뜻하는 속성(필드)과, 행위를 정의하는 프로그램코드인 메서드로 추상화한 사용자 정의 자료형의 개념
타입 = 속성 + 행위

class 클래스_명 {

// 필드 정의 (속성, 멤버 변수)

// 메서드 정의(행위, 프로그램 코드)

}
